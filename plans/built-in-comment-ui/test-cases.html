1. Interactive Playground

Exercise the composer trigger without a backend. Change role / panel / intent, select text in the mock doc, and copy the payload the client would POST to /api/comments/:scenario/:doc (see Design §5).

The author flow ends when the server appends the JSONL line and returns 201. The author value is stamped server-side from req.user.name; the client cannot set it.

Anchors use prefix + exact + suffix inside a stable sec-<hex> section id so they survive regen. A three-tier cascade (exact, exact-in-section, fuzzy) picks the best hold; leftovers become orphans.

12. Cross-cutting Battery

Security payload lists

AC-5 path-traversal vectors (≥10): ../etc/passwd, ..%2Fetc, ..%252Fetc, plan/../.., plan..\etc (Windows sep), plan%00.. (null byte), plan/. + symlink target, absolute /etc/passwd, unicode slash \u2215, overly long segment >80 chars, segment containing space. All expect 400 before any fs.open.

AC-6 XSS vectors (≥8): <img src=x onerror=alert(1)>, <svg onload=alert(1)>, <iframe src=javascript:...>, <a href="javascript:alert(1)">link</a>, entity-encoded &lt;script&gt;, polyglot "><svg/onload=alert(1)>, unicode RTL override \u202e, template literal ${alert(1)}. All render as text.

Accessibility battery

  • Keyboard-only round trip: Tab to doc → select → compose → submit → close — no mouse.
  • Focus never leaves composer on Tab (focus trap); Esc returns focus to anchor.
  • ARIA live region announces "Comment posted" within 1s of 201.
  • High-contrast + dark-mode both pass axe-core (0 violations on widget surfaces).

Performance budgets

  • TTFC (time-to-first-comment) p50 < 30s; p95 < 90s from landing on login (Design §1 success metric).
  • Anchor scan on load: <50ms for ≤100 comments on a 6k-word doc; <200ms for 500 comments.
  • SSE broadcast fan-out latency: ≤500ms localhost / ≤1.5s devtunnel.
  • [ ] TODO: confirm 500-comment anchor scan budget with backend dev under real JSONL sizes.
