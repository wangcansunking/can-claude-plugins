L1. Proposal Accept — anchor-drift guard rejects valid proposal (RESOLVED)

Symptom
Clicking Accept & apply in the proposal modal surfaced “anchor no longer matches; reattach manually” below the diff. Server returned 409 ANCHOR_DRIFT. Doc was not rewritten.
Root cause
plan-harness/local-proxy/src/revise-dispatcher.js:151-156. acceptProposal did html.includes(anchor.exact) and html.includes(parsed.from) against the raw HTML source. The anchor's exact field captures the rendered plain text (e.g. Phase 5 — composer panel submits with Ctrl+Enter), but the on-disk HTML contains entity-encoded em-dash (&mdash;) and inline tags (<kbd>Ctrl</kbd>+<kbd>Enter</kbd>). The naive substring check never matched.
Fix applied
Added renderText(html) in revise-dispatcher.js. It walks the raw HTML and emits (a) a rendered-text view with tags stripped + common entities decoded + whitespace collapsed, plus (b) a per-char index map back to the raw source and (c) a per-char raw-length array. acceptProposal now:
  • normalizes anchor.exact and parsed.from with normalizeSpaces;
  • does indexOf against the rendered-text view (so &mdash; matches —, and text inside <kbd> wrappers matches);
  • refuses when the needle appears more than once (new AMBIGUOUS_TARGET 409);
  • maps match start + end back to raw-HTML offsets via the position map and splices parsed.to at those offsets (preserving surrounding tags and entities).
Regression guard
New smoke-test block [revise] acceptProposal handles entities + inline tags in scripts/smoke-test.mjs: scaffolds a temp workspace with realistic HTML (&mdash; + <kbd>), seeds a proposal that targets the rendered text, calls acceptProposal, and asserts (1) doc is rewritten, (2) original line is not duplicated, (3) revise: accepted event appended, (4) resolve: true event appended. Total suite is now 89 assertions (was 85) — all green.
